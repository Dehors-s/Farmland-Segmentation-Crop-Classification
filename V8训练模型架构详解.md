## 0. 前置知识：图像分割要解决什么问题？

### 语义分割（Semantic Segmentation）

**定义：** 给图像的**每个像素**打一个类别标签。

对于耕地地块分割：
- **输入**：遥感影像（RGB 或 RGBN 多光谱）
- **输出**：每个像素是"耕地"还是"非耕地"的二值图（二分类语义分割）
- **难点**：
  - 地块边界模糊（耕地在遥感影像上边界不清晰）
  - 形状不规则（农田不是规整的矩形）
  - 地块之间紧邻（相邻地块之间只有一条缝）

### 为什么是U-Net？

**U-Net** 最初是为医学图像分割设计的（Ronneberger et al., 2015），它的名字来源于网络形状像字母"U"。后来被遥感领域广泛采用，核心设计理念：

```
编码器（下采样） → 提取语义特征、压缩空间信息
解码器（上采样） → 恢复空间分辨率
跳连（Skip Connection） → 把编码器的细节信息传给解码器
```

**关键洞察：** 分割任务的输出需要"像素级"精度，仅靠编码器-解码器结构（Autoencoder）会丢失空间细节——跳连就是用来补细节的。

### 语义分割 vs 实例分割 vs 全景分割

| 任务 | 定义 | 例子 |
|------|------|------|
| **语义分割** | 给每个像素分**类别**，同类别不分个体 | 所有耕地像素标为1，其他标为0 |
| **实例分割** | 给每个像素分**类别+个体ID** | 地块A的像素标为1，地块B的像素标为2 |
| **全景分割** | 语义+实例的结合 | 耕地地块每个个体不同ID，背景按类别 |

V8做的是**语义分割**（二分类），后处理通过连通域分析转换为实例分割。

## 1. V8总体架构总览

V8模型是一个 **多任务 CBAM-U-Net**，专门针对**4通道多光谱（R+G+B+NIR）遥感影像的耕地地块分割**设计。

### 一句话概括

> 用ResNet做编码器提取多尺度特征 → 用带CBAM注意力的解码器逐步恢复分辨率 → 用三个任务头分别预测掩膜、边界、距离 → 融合三个结果得到最终分割。

### 完整架构图

```
输入: [B, 4, H, W] RGBN遥感影像 (B=batch, 4=通道, H=W=512)
  │
  ├─ [SpectralAttention] ──── 轻量光谱通道重标定
  │
  ├─ [CBAMUNet 主干]
  │   │
  │   ├─ Encoder (ResNet50/34/18):
  │   │   stage1: conv1(7×7, s2) → BN → ReLU → MaxPool(3×3, s2)   [H/4 × H/4,  C1]
  │   │   stage2: layer1 (多个Bottleneck)                           [H/4 × H/4,  C2]
  │   │   stage3: layer2                                           [H/8 × H/8,  C3]
  │   │   stage4: layer3                                           [H/16 × H/16, C4]
  │   │   stage5: layer4                                           [H/32 × H/32, C5]
  │   │
  │   ├─ Bridge: Conv3×3 → BN → ReLU                               [H/32 × H/32, 256]
  │   │
  │   └─ Decoder (4级 CBAMDecoderBlock):
  │       dec4: ConvT(2×2) ↑2x → Cat(e4) → Conv → CBAM → Conv     [H/16, 128]
  │       dec3: ConvT(2×2) ↑2x → Cat(e3) → Conv → CBAM → Conv     [H/8,  64]
  │       dec2: ConvT(2×2) ↑2x → Cat(e2) → Conv → CBAM → Conv     [H/4,  32]
  │       dec1: ConvT(2×2) ↑2x → Cat(e1) → Conv → CBAM → Conv     [H/2,  32]
  │
  └─ [MultiTaskUNet 三任务头] (在dec1的32ch特征之上)
      │
      ├─ BoundaryHead: Conv3×3→16 → BN+ReLU+Drop → ConvT→16 → Conv1×1→1
      ├─ DistanceHead: Conv3×3→16 → BN+ReLU+Drop → ConvT→16 → Conv1×1→1 → Sigmoid
      └─ SegUp: ConvT(2×2)→16 → BN+ReLU+Drop
           │
           └─ Fusion: Cat(16ch_seg, 1ch_boundary, 1ch_distance) → Conv3×3→16
                      → BN+ReLU+Drop → Conv1×1→classes
```

### 补充知识：B,H,W,C 维度约定

深度学习中的数据通常是四维张量（Tensor）：

```
[B, C, H, W] 或 [B, H, W, C]
```

- **B (Batch)**：一次输入多少张图片（V8默认 batch_size=16）
- **C (Channel)**：图像的通道数（RGB=3, RGBN=4）
- **H (Height)**：图像高度
- **W (Width)**：图像宽度

V8使用的是 `[B, C, H, W]` 格式（PyTorch默认格式）。

## 2. 输入层：RGBN多光谱 + SpectralAttention

### 2.1 多光谱输入

```python
# 命令行参数
--in_channels 4
```

遥感影像通常有4个波段：

| 波段 | 波长范围 | 用途 |
|------|---------|------|
| R (红) | 620-750nm | 植被吸收波段，区分植被/非植被 |
| G (绿) | 495-570nm | 植被反射峰，人眼敏感 |
| B (蓝) | 450-495nm | 大气散射强，用于大气校正 |
| NIR (近红外) | 750-900nm | **植被强反射**，耕地识别的关键波段 |

**为什么NIR重要？** 健康植被在近红外波段反射率远高于其他地物，这是**归一化植被指数（NDVI）** 的基础：$NDVI = \frac{NIR - R}{NIR + R}$。

### 2.2 自适应首层卷积

ResNet的conv1原本只接受3个通道（RGB），V8将其适配到4通道：

```python
if in_channels != 3:
    old_conv1 = base_model.conv1
    new_conv1 = nn.Conv2d(in_channels, ...)   # 创建4通道的新卷积
    new_conv1.weight[:, :3] = old_conv1.weight # RGB部分复制预训练权重
    mean_weight = old_conv1.weight.mean(dim=1, keepdim=True)
    for idx in range(3, in_channels):          # NIR部分用RGB权重均值初始化
        new_conv1.weight[:, idx:idx+1] = mean_weight
```

**补充知识：迁移学习（Transfer Learning）**

迁移学习的核心思想：在一个大数据集（如ImageNet，1400万张图片）上预训练的模型，在小数据集上微调也能取得好效果。

- **为什么有效**：CNN的低层特征（边缘、纹理、颜色斑点）是通用的，无论任务是什么
- **V8的做法**：在ImageNet上预训练的ResNet基础上，把conv1扩展为4通道，其他层保持不变
- **NIR初始化的道理**：用RGB三通道权重的均值初始化NIR通道，相当于"不知道NIR和哪个可见光波段更相关，就用平均值"

### 2.3 SpectralAttention

```python
class SpectralAttention(nn.Module):
    """轻量光谱注意力：对多通道输入进行通道重标定。"""
    def __init__(self, in_channels, reduction_ratio=8):
        super().__init__()
        hidden = max(4, in_channels // reduction_ratio)
        self.pool = nn.AdaptiveAvgPool2d(1)
        self.proj = nn.Sequential(
            nn.Conv2d(in_channels, hidden, kernel_size=1, bias=False),
            nn.ReLU(inplace=True),
            nn.Conv2d(hidden, in_channels, kernel_size=1, bias=False),
            nn.Sigmoid(),
        )

    def forward(self, x):
        w = self.proj(self.pool(x))  # 全局平均池化 → MLP → Sigmoid → 权重
        return x * w                 # 权重逐通道乘原始特征
```

**这个模块做什么？**
- 对4个光谱通道做一个**全局的"重要性打分"**
- 每个通道乘以一个权重，告诉模型"NIR通道更重要"或"RGB更重要"
- 权重是根据**整张图像的内容动态计算**的

**工作流程：**
```
输入 x: [B, 4, H, W]
  │
  ▼ AdaptiveAvgPool2d(1): 每个通道取全局平均
  [B, 4, 1, 1]
  │
  ▼ Conv1×1 → ReLU: 压缩到 hidden 维
  [B, hidden, 1, 1]
  │
  ▼ Conv1×1 → Sigmoid: 恢复到4个通道，值域[0,1]
  [B, 4, 1, 1]  ← 这就是4个通道的权重
  │
  ▼ x * w: 每个通道乘以对应权重
  [B, 4, H, W]
```

**补充知识：通道注意力（Channel Attention）**

SpectralAttention本质是**SENet（Squeeze-and-Excitation Network）** 的简化版（Hu et al., 2018, ImageNet冠军）：

SENet的完整设计：
```
Squeeze: 全局平均池化 (压缩全局空间信息)
Excitation: FC → ReLU → FC → Sigmoid (学习通道间关系)
Scale: 权重 × 原始特征 (重标定)
```

为什么叫"注意力"？因为机制类似人的注意力——有选择地关注重要信息，忽视不重要信息。

### 3.2 什么是残差块（Residual Block）

**标准卷积块的局限：**

传统的深度网络叠层是这样：

```
输入 x → Conv → BN → ReLU → Conv → BN → ReLU → 输出
```

问题是：层数越深→梯度越小→网络退化（Degradation），即更深的网络反而效果更差。

**残差块的核心思想：**

```
        输入 x
          │
     ┌────┴────┐
     │ Conv3×3 │
     │   BN    │
     │  ReLU   │
     │ Conv3×3 │
     │   BN    │
     └────┬────┘
          │  (+)
     y = F(x) + x   ← 关键！
          │
         ReLU
          │
         输出 y
```

公式：$y = \mathcal{F}(x, \{W_i\}) + x$

其中 $\mathcal{F}(x, \{W_i\})$ 是两个卷积层学到的"残差映射"。

**为什么残差连接解决了退化问题？**

1. **梯度高速公路**：反向传播时，梯度可以通过 `+x` 的恒等连接直接回传到前面层，不会衰减
2. **学习残差更容易**：$\mathcal{F}(x) = y - x$ 比直接学习 $y$ 更容易——因为 $y$ 往往和 $x$ 相似
3. **直觉理解**：残差块相当于说"我不需要从头学输出，我只需要学输出和输入的差别"

### 3.3 Bottleneck块（ResNet50特有）

ResNet50/101/152用的是Bottleneck块，比BasicBlock（ResNet18/34）参数量更少、深度更深：

```
Bottleneck结构:
输入: 256ch
  │
  Conv1×1, 64ch   ← 降维（压缩通道数）
  BN, ReLU
  │
  Conv3×3, 64ch   ← 空间特征提取（在低维空间做，计算量小）
  BN, ReLU
  │
  Conv1×1, 256ch  ← 升维（恢复通道数）
  BN
  │ (+)  ← 与输入相加
  ReLU
  │
输出: 256ch
```

**补充知识：为什么Bottleneck高效？**

普通3×3卷积的计算量：$C_{in} \times C_{out} \times K^2 = 256 \times 256 \times 9 = 589,824$

Bottleneck的计算量：$256 \times 64 \times 1 + 64 \times 64 \times 9 + 64 \times 256 \times 1 = 16,384 + 36,864 + 16,384 = 69,632$

**节省了约8倍计算量！**

## 4. Bridge桥接层

```python
self.bridge = nn.Sequential(
    nn.Conv2d(encoder_channels[-1], decoder_channels[0], 3, padding=1),
    nn.BatchNorm2d(decoder_channels[0]),
    nn.ReLU(inplace=True),
)
```

**作用：** 将编码器输出的2048维特征压缩到解码器需要的256维。

```
encoder5: [B, 2048, 8, 8]  →  Bridge  →  [B, 256, 8, 8]
```

**补充知识：1×1 vs 3×3卷积**

| 卷积核 | 作用 | 计算量 |
|--------|------|-------|
| **1×1** | 只改变通道数（通道间的线性组合），不改变空间分辨率 | 小 |
| **3×3** | 同时改变通道数和提取空间特征（感受野3×3） | 大 |
| **7×7** | 更大的感受野，更全局的特征 | 更大 |

Bridge使用3×3而非1×1，说明它在通道压缩的同时还想提取一些空间特征。

## 5. 解码器：CBAMDecoderBlock

### 5.1 单个DecoderBlock的结构

```python
class CBAMDecoderBlock(nn.Module):
    def forward(self, x, skip):
        x = self.up(x)                    # ① 上采样: 2x2 ConvTranspose, stride=2
        x = torch.cat([x, skip], dim=1)   # ② 拼接跳连
        x = self.relu1(self.bn1(self.conv1(x)))  # ③ Conv3x3 + BN + ReLU
        x = self.dropout1(x)              # ④ Dropout
        if self.use_cbam:
            x = self.cbam(x)              # ⑤ CBAM注意力
        x = self.relu2(self.bn2(self.conv2(x)))  # ⑥ Conv3x3 + BN + ReLU
        x = self.dropout2(x)              # ⑦ Dropout
        return x
```

### 5.2 转置卷积（Transposed Convolution）

**最直观的理解：**

| 操作 | 输入尺寸 | 输出尺寸 | 作用 |
|------|---------|---------|------|
| 普通卷积（Conv） | 大 | 小 | 下采样（提取特征、压缩） |
| 转置卷积（ConvTranspose） | 小 | 大 | 上采样（恢复分辨率） |

**转置卷积工作原理（kernel=2, stride=2）：**

```
输入 2×2 → 每个像素周围插0 → 3×3 → 卷积 → 输出 4×4

输入: [a b]    插0后: [0 0 0 0]    卷积核: [w1 w2]
      [c d]           [0 a b 0]            [w3 w4]
                      [0 c d 0]
                      [0 0 0 0]
```

**不是简单的"逆卷积"！** 转置卷积只是"形状上是卷积的逆操作"，参数是学出来的，不是卷积的数学逆运算。

### 5.3 跳连（Skip Connection）的作用

```
Decoder的上采样过程会丢失细节信息（因为转置卷积类似于"插值+学习"）
Encoder保留着各层级的细节特征（因为还没下采样那么多）

跳连 = 把编码器对应层的细节特征 传给 解码器 → 精确定位
```

**没有跳连会怎样？**

U-Net退化为Autoencoder：

```
Autoencoder: 编码器压缩图像 → 解码器从压缩特征重建
             问题：压缩时丢失的空间细节无法恢复
             
U-Net:       编码器压缩 + 跳连提供细节
             结果：语义信息来自编码器深层，位置信息来自编码器浅层
```

**V8的四级跳连：**

```
dec4: bridge(8×8) + encoder4(16×16)  →  16×16  (大尺度语义+中尺度细节)
dec3: dec4(16×16) + encoder3(32×32)  →  32×32
dec2: dec3(32×32) + encoder2(64×64)  →  64×64
dec1: dec2(64×64) + encoder1(128×128) → 128×128 (小尺度语义+高分辨率细节)
```

每一级跳连补充不同尺度的细节，最终dec1的输出同时包含全局语义和局部细节。

### 5.4 Dropout2d

```python
self.dropout1 = nn.Dropout2d(dropout_rate)  # dropout_rate = 0.2
```

**Dropout是什么？** 训练时随机"丢弃"（置为0）一部分神经元，防止过拟合。

**Dropout2d vs Dropout：**
- `nn.Dropout`：随机丢弃**单个元素**
- `nn.Dropout2d`：随机丢弃**整个通道**（特征图）

对于CNN，相邻像素高度相关，丢弃单个像素效果差。丢弃整个通道更有效——迫使网络不依赖于某个特定通道的特征。

**补充知识：为什么Dropout能防止过拟合？**

1. **集成学习效果**：每次训练相当于训练了一个不同的子网络，测试时相当于集成了所有子网络
2. **减弱神经元共适应**：神经元不能依赖其他特定神经元的存在，必须自己学会提取有用特征
3. **正则化**：相当于L2正则化的一种近似

## 6. CBAM注意力模块详解

### 6.1 什么是注意力机制？

**注意力（Attention）** 受人类视觉启发的机制：

> 人看一张图片时，不会均匀关注每个像素，而是**有选择地关注重要区域**

在深度学习中，注意力机制就是**让模型学会"应该关注哪里"**。

CBAM（Convolutional Block Attention Module, Woo et al., 2018）包含两个子模块：

### 6.2 ChannelAttention（通道注意力）

**回答的问题：** "哪些通道更重要？"

```python
class ChannelAttention(nn.Module):
    def __init__(self, in_channels, reduction_ratio=16):
        super().__init__()
        hidden = max(8, in_channels // reduction_ratio)
        self.avg_pool = nn.AdaptiveAvgPool2d(1)
        self.max_pool = nn.AdaptiveMaxPool2d(1)
        self.fc = nn.Sequential(
            nn.Conv2d(in_channels, hidden, 1, bias=False),
            nn.ReLU(inplace=True),
            nn.Conv2d(hidden, in_channels, 1, bias=False),
        )
        self.sigmoid = nn.Sigmoid()

    def forward(self, x):
        avg_out = self.fc(self.avg_pool(x))    # ① 全局平均池化 → FC
        max_out = self.fc(self.max_pool(x))    # ② 全局最大池化 → FC
        return self.sigmoid(avg_out + max_out) # ③ 相加 → Sigmoid
```

**工作流程：**

```
输入 x: [B, C, H, W]
  │
  ├─ AdaptiveAvgPool2d(1): [B, C, 1, 1]   ← 每个通道取全局均值
  ├─ AdaptiveMaxPool2d(1): [B, C, 1, 1]   ← 每个通道取全局最大值
  │
  ├─ 共享MLP:
  │   Conv1×1 → C/r维度 → ReLU → Conv1×1 → C维度
  │
  ├─ 相加 avg_out + max_out
  ├─ Sigmoid → [B, C, 1, 1]   ← C个通道的权重
  │
  ▼ x * 通道权重
  [B, C, H, W]
```

**直觉理解：**

- **平均池化**：捕获通道的"整体响应"
- **最大池化**：捕获通道的"最强响应"
- **两者互补**：一个看平均，一个看峰值
- **共享MLP**：学习通道之间的交互关系
- **Sigmoid**：把分数映射到[0,1]区间，作为权重

**补充知识：为什么通道注意力有效？**

在CNN中，不同通道检测不同特征：
- 通道1：检测水平边缘
- 通道2：检测红色区域
- 通道3：检测纹理

对于某张具体图片，有些通道比其他通道更重要。通道注意力就是让模型**动态调整各通道的贡献**。

### 6.3 SpatialAttention（空间注意力）

**回答的问题：** "图像中哪些位置更重要？"

```python
class SpatialAttention(nn.Module):
    def __init__(self, kernel_size=7):
        super().__init__()
        padding = kernel_size // 2
        self.conv = nn.Conv2d(2, 1, kernel_size, padding=padding, bias=False)
        self.sigmoid = nn.Sigmoid()

    def forward(self, x):
        avg_out = torch.mean(x, dim=1, keepdim=True)  # ① 通道维取均值
        max_out, _ = torch.max(x, dim=1, keepdim=True) # ② 通道维取最大值
        x_cat = torch.cat([avg_out, max_out], dim=1)   # ③ 拼接: [B, 2, H, W]
        return self.sigmoid(self.conv(x_cat))          # ④ 7×7 Conv → Sigmoid
```

**工作流程：**

```
输入 x: [B, C, H, W]
  │
  ├─ mean(dim=1): [B, 1, H, W]   ← 每个位置取所有通道的平均值
  ├─ max(dim=1):  [B, 1, H, W]   ← 每个位置取所有通道的最大值
  │
  ├─ Cat: [B, 2, H, W]           ← 拼接成双通道特征图
  ├─ Conv7×7: [B, 1, H, W]       ← 学习空间位置相关性
  ├─ Sigmoid: [B, 1, H, W]       ← 每个位置的权重
  │
  ▼ x * 空间权重
  [B, C, H, W]
```

**为什么用7×7卷积？** 空间注意力需要**局部上下文**。7×7意味着每个位置的权重由它周围7×7区域的特征共同决定。

### 6.4 CBAM = 通道注意力 + 空间注意力

```python
class CBAM(nn.Module):
    def forward(self, x):
        x = self.channel_attention(x) * x   # 先做通道注意力
        x = self.spatial_attention(x) * x   # 再做空间注意力
        return x
```

**串行设计的原因：**

> 论文实验证明：通道注意 → 空间注意（串行）> 空间注意 → 通道注意 > 并行

- **通道注意力**先回答"什么重要"
- **空间注意力**再回答"哪里重要"
- 顺序很重要：先知道看什么特征，再知道去哪里找

**类比理解：**

> 你在找一栋红色房子（通道注意力关注"红色"特征）→ 然后你在图像中找红色区域（空间注意力关注"红色区域"位置）

### 6.5 CBAM在V8中的位置

CBAM嵌入在解码器的每个CBAMDecoderBlock中：

```
DecoderBlock:
  Conv1 → BN → ReLU → Dropout
     ↓
   CBAM  ← 放在第一个卷积之后、第二个卷积之前
     ↓
  Conv2 → BN → ReLU → Dropout
```

**为什么放在这个位置？** 第一个卷积提取特征 → CBAM增强重要特征抑制不重要特征 → 第二个卷积进一步处理增强后的特征。

## 7. 三任务头与多任务学习

### 7.1 为什么需要多任务学习？

**单任务问题：** 只做分割（掩膜预测），模型容易忽略边界信息。

**多任务学习（Multi-Task Learning）的定义：**
> 一个深度学习模型通过**特征共享**的方式，同时完成**多个相关任务**。

V8的三个任务：

| 任务 | 类型 | 输出 | 作用 |
|------|------|------|------|
| **掩膜分割**（主任务） | 二分类 | 耕地/非耕地 | 最终目标 |
| **边界预测**（辅助任务） | 二分类 | 边界/非边界 | 强化边缘特征 |
| **距离回归**（辅助任务） | 回归 | 像素到边界的距离 | 强化内部一致性 |

### 7.2 三个任务头的实现

所有三个任务头共享同一个解码器输出的32通道特征图。

#### 边界预测头（BoundaryHead）

```python
self.boundary_head = nn.Sequential(
    nn.Conv2d(32, 16, kernel_size=3, padding=1),  # 32→16降维
    nn.BatchNorm2d(16),
    nn.ReLU(inplace=True),
    nn.Dropout2d(dropout_rate),
    nn.ConvTranspose2d(16, 16, kernel_size=2, stride=2),  # 上采样×2
    nn.Conv2d(16, 1, kernel_size=1),  # 输出单通道边界图
)
```

- 输入：`[B, 32, 128, 128]`（以512输入为例，解码器输出H/4）
- 输出：`[B, 1, 256, 256]`（上采样到H/2）
- 不做Sigmoid，直接用logits计算损失（BCEWithLogitsLoss内部会做Sigmoid）

#### 距离预测头（DistanceHead）

```python
self.distance_head = nn.Sequential(
    nn.Conv2d(32, 16, kernel_size=3, padding=1),
    nn.BatchNorm2d(16),
    nn.ReLU(inplace=True),
    nn.Dropout2d(dropout_rate),
    nn.ConvTranspose2d(16, 16, kernel_size=2, stride=2),
    nn.Conv2d(16, 1, kernel_size=1),
    nn.Sigmoid(),  # 输出范围[0,1]
)
```

与边界头的区别：最后加了Sigmoid，因为距离是归一化之后的[0,1]区间值。

**距离图如何生成（在数据集类中）：**

```python
def generate_distance_map(mask):
    mask_uint8 = (mask * 255).astype(np.uint8)
    dist_transform = cv2.distanceTransform(mask_uint8, cv2.DIST_L2, cv2.DIST_MASK_PRECISE)
    max_dist = dist_transform.max()
    dist_normalized = dist_transform / max_dist if max_dist > 0 else dist_transform
    return dist_normalized
```

- 对mask中的每个"耕地"像素，计算它到最近"非耕地"像素的欧氏距离
- 归一化到[0,1]，1表示离边界最远（地块中心），0表示边界上

#### 分割特征上采样（SegUp）

```python
self.seg_up = nn.Sequential(
    nn.ConvTranspose2d(32, 16, kernel_size=2, stride=2),
    nn.BatchNorm2d(16),
    nn.ReLU(inplace=True),
    nn.Dropout2d(dropout_rate),
)
```

- 输出：`[B, 16, 256, 256]`
- 不直接预测，而是作为**融合特征**

### 7.3 多任务学习的三大好处

| 好处 | 解释 | 类比 |
|------|------|------|
| **特征共享** | 边界任务教模型关注边缘→分割任务受益 | 学画画和学书法共享"手部控制"能力 |
| **正则化** | 辅助任务防止过拟合到单一任务的噪声 | 同时练长跑和举重比只练一项更平衡 |
| **归纳偏置** | 告诉模型"地块有边界，边界是重要的" | 先学"物体有边缘"再学"这是什么物体" |

### 7.4 边界目标生成函数

```python
@staticmethod
def get_boundary_targets(masks):
    kernel_size = 3
    padding = kernel_size // 2
    eroded = -F.max_pool2d(-masks, kernel_size, stride=1, padding=padding)  # 腐蚀
    dilated = F.max_pool2d(masks, kernel_size, stride=1, padding=padding)   # 膨胀
    return dilated - eroded  # 形态学梯度 = 膨胀 - 腐蚀 = 边界
```

**形态学梯度（Morphological Gradient）原理：**

```
原始mask:      ████████      ████████
腐蚀(erode):    ██████        ██████      ← 向内收缩1像素
膨胀(dilate):  ██████████    ██████████  ← 向外扩张1像素
梯度(diff):      ██  ██        ██  ██    ← 膨胀-腐蚀 = 边界
```

为什么不是直接用Canny边缘检测？因为形态学梯度保证边界是**闭合的**，且与mask完全对齐。

## 8. 融合模块（Fusion）

```python
self.fusion = nn.Sequential(
    nn.Conv2d(16 + 1 + 1, 16, kernel_size=3, padding=1),  # 18通道 → 16通道
    nn.BatchNorm2d(16),
    nn.ReLU(inplace=True),
    nn.Dropout2d(dropout_rate),
    nn.Conv2d(16, classes, kernel_size=1),  # 16通道 → 1通道(二分类)
)
```

**融合过程：**

```
seg_feat:     [B, 16, H, W]   ← 分割特征
boundary:     [B, 1, H, W]    ← 边界概率
distance:     [B, 1, H, W]    ← 距离值
  │
  ▼ Cat(dim=1)
[B, 18, H, W]
  │
  ▼ Conv3×3 → BN → ReLU → Dropout
[B, 16, H, W]
  │
  ▼ Conv1×1
[B, 1, H, W]   ← 最终分割图（logits值，后续加Sigmoid）
```

**为什么这样融合？**

1. **边界信息帮助分割**：知道边界在哪里，就知道在哪里"切开"相邻地块
2. **距离信息帮助分割**：知道某个像素离边界多远，有助于判断它属于哪个地块
3. **可学习的融合**：不是简单相加，而是让模型自己学"怎么结合这三个信息"

**补充知识：注意力融合 vs 特征融合**

这里用**拼接（Concatenation）+ 卷积**的方式融合，是最常见的特征融合方法。还有其他的融合方式：

| 融合方式 | 做法 | 特点 |
|---------|------|------|
| **拼接+卷积** | Cat → Conv（V8的方法） | 灵活但参数多 |
| **逐元素相加** | a + b | 简单，要求输入通道数一致 |
| **加权求和** | α×a + β×b | 权重要学 |
| **注意力门控** | gating机制 | 更复杂的交互 |

## 9. 损失函数体系

### 9.1 分割损失：CombinedLoss

```python
self.criterion_seg = CombinedLoss(
    loss_type="log_cosh_dice",
    dice_weight=0.5,
    aux_weight=0.5,
)
```

`CombinedLoss` = 主损失（Primary）× 0.5 + 辅助损失（Aux）× 0.5

#### 主损失：LogCoshDiceLoss

```python
class LogCoshDiceLoss(nn.Module):
    def forward(self, inputs, targets, smooth=1):
        inputs_sigmoid = torch.sigmoid(inputs)
        dice = (2.0 * intersection + smooth) / (inputs_flat.sum() + targets_flat.sum() + smooth)
        return torch.log(torch.cosh(1 - dice))
```

**Dice系数基础：**

$Dice = \frac{2|A \cap B|}{|A| + |B|}$

- A = 预测为正样本的像素集合
- B = 真实为正样本的像素集合
- 范围 [0, 1]，1表示完美重叠

**Dice Loss：** $Loss_{Dice} = 1 - Dice$

**Log-Cosh改进：**

$Loss_{LogCoshDice} = \log(\cosh(1 - Dice))$

其中 $\cosh(x) = \frac{e^x + e^{-x}}{2}$

**Log-Cosh的优点：**

```
当 x→0 (Dice接近1):  cosh(x) ≈ 1 + x²/2  →  log(cosh(x)) ≈ x²/2  (类似L2)
当 x→∞ (Dice接近0):  cosh(x) ≈ eˣ/2      →  log(cosh(x)) ≈ x - log2  (类似L1)

→ 小误差时像L2（平滑收敛），大误差时像L1（对异常值鲁棒）
→ 比普通Dice Loss更平滑，梯度更稳定
```

**补充知识：常见损失函数对比**

| 损失函数 | 特点 | 适合场景 |
|---------|------|---------|
| **BCE Loss** | 像素级独立优化 | 类别平衡 |
| **Dice Loss** | 关注整体重叠，擅长处理不平衡 | 医学/遥感分割（前景小） |
| **Focal Loss** | 放大难分样本的梯度 | 类别极度不平衡 |
| **Tversky Loss** | 可调节FP/FN权重 | 需要偏重召回率或精确率 |
| **Log-Cosh Dice** | Dice的平滑版，梯度稳定 | 通用推荐（V8默认） |

#### 辅助损失：FocalLoss

```python
class FocalLoss(nn.Module):
    def __init__(self, alpha=0.25, gamma=2):
        ...
    def forward(self, inputs, targets):
        bce = F.binary_cross_entropy_with_logits(inputs, targets, reduction="none")
        pt = torch.exp(-bce)
        focal_loss = self.alpha * (1 - pt) ** self.gamma * bce
        return focal_loss.mean()
```

**Focal Loss公式：** $FL(p_t) = -\alpha(1-p_t)^\gamma \log(p_t)$

其中 $p_t$ 是模型对正确类别的预测概率。

**直观解释（γ=2为例）：**

```
如果模型对某像素预测p=0.9（高置信度、容易样本）:
  (1-0.9)² = 0.01  → 权重很小
  该样本对损失贡献很小

如果模型预测p=0.3（低置信度、困难样本）:
  (1-0.3)² = 0.49  → 权重很大
  该样本对损失贡献大
```

**为什么组合LogCoshDice和Focal？**

- LogCoshDice：从**全局重叠**角度优化（宏观好）
- FocalLoss：从**困难像素**角度优化（细节好）
- 组合 = 全局 + 细节

### 9.2 边界损失：DiceBCELoss

```python
class DiceBCELoss(nn.Module):
    def forward(self, inputs, targets, smooth=1):
        dice_loss = 1 - (2.0 * intersection + smooth) / (inputs_flat.sum() + targets_flat.sum() + smooth)
        bce_loss = F.binary_cross_entropy_with_logits(inputs, targets, reduction="mean")
        return dice_loss + bce_loss
```

Dice + BCE的互补：

| 组件 | 优化目标 | 缺点 |
|------|---------|------|
| Dice | 整体重叠率 | 梯度不稳定（分母小时震荡） |
| BCE | 逐个像素正确 | 类别不平衡时偏向多数类 |
| 组合 | **既看整体又看局部** | 互相弥补 |

### 9.3 距离损失：L1Loss

```python
self.criterion_dist = nn.L1Loss()  # 平均绝对误差
```

**MAE vs MSE：**

| 损失 | 公式 | 对异常值 |
|------|------|---------|
| **MAE (L1)** | $|y - \hat{y}|$ | 鲁棒（线性惩罚） |
| **MSE (L2)** | $(y - \hat{y})^2$ | 敏感（平方惩罚） |

距离值在[0,1]区间，用MAE更合适——避免对错误预测的距离值过度惩罚。

### 9.4 总损失函数

```python
# 动态边界权重
dynamic_boundary_weight = config["boundary_weight"]  # 初始=1.0
if epoch > 10:
    dynamic_boundary_weight *= 0.5
if epoch > 20:
    dynamic_boundary_weight *= 0.5
dynamic_boundary_weight = max(dynamic_boundary_weight, min_boundary_weight)  # 下限0.3

# 总损失
loss = loss_seg + dynamic_boundary_weight * loss_bdy + 0.25 * loss_dist
```

| 时期 | 边界权重 | 含义 |
|------|---------|------|
| epoch 1-10 | 1.0 | 边界任务全力辅助 |
| epoch 11-20 | 0.5 | 边界任务逐渐退场 |
| epoch 21+ | 0.25 (≥0.3) | 边界任务仅作微调 |

**为什么边界权重衰减？**

- 训练初期：模型还不会分割，边界任务帮它"定位边缘在哪里"
- 训练后期：模型已经学会了边界特征，过度强调边界反而会干扰分割的精细调整

## 10. 训练策略

### 10.1 优化器：AdamW

```python
self.optimizer = optim.AdamW(self.model.parameters(), lr=lr, weight_decay=1e-4)
```

**Adam与AdamW的区别：**

| 优化器 | 权重衰减实现 | 效果 |
|--------|------------|------|
| Adam | 耦合在学习率自适应中 | 效果较差 |
| AdamW | 解耦权重衰减（Decoupled Weight Decay） | 效果更好，泛化更强 |

**补充知识：权重衰减（Weight Decay）**

$Loss = Loss_{data} + \frac{\lambda}{2} ||W||^2$

给大的权重加惩罚，迫使模型学习更简单的模式，防止过拟合。

### 10.2 学习率调度

**阶段1：Warmup（前2-3 epoch）**

```python
warmup_scheduler = torch.optim.lr_scheduler.LinearLR(
    self.optimizer, start_factor=0.1, total_iters=self.warmup_epochs
)
```

- 学习率从 $0.1 \times target\_lr$ 线性增加到 $target\_lr$
- **目的**：避免初期大学习率导致梯度爆炸

**阶段2：ReduceLROnPlateau**

```python
main_scheduler = optim.lr_scheduler.ReduceLROnPlateau(
    self.optimizer, mode="max", factor=0.5, patience=3, min_lr=min_lr
)
```

- 如果验证IoU连续3轮没有提升 → 学习率×0.5
- **目的**：接近最优时缩小步长，精细调优

### 10.3 梯度裁剪

```python
torch.nn.utils.clip_grad_norm_(self.model.parameters(), max_norm=1.0)
```

- 把梯度的L2范数限制在1.0以内
- **目的**：防止梯度爆炸（一个batch的梯度太大导致参数更新失控）

### 10.4 混合精度训练（AMP）

```python
self.scaler = GradScaler()
with autocast():
    seg_logits, bdy_logits, dist_logits = self.model(imgs)
    ...
self.scaler.scale(loss).backward()
self.scaler.step(self.optimizer)
self.scaler.update()
```

- 前向传播用**float16**（速度快、省显存）
- 梯度用**float32**（保持精度）
- `GradScaler`防止float16下小梯度"下溢"为0

**效果：显存节省约40%，速度提升约1.5-2倍。**

### 10.5 早停（Early Stopping）

```python
if patience_counter >= early_stopping_patience:  # V8: 8轮, V8_4090: 12轮
    break
```

- 验证IoU连续N轮不提升 → 停止训练
- **目的**：防止过拟合，节省时间

## 11. 数据流全景图解

以输入 512×512×4 为例，完整的数据流：

```
输入: [B, 4, 512, 512] RGBN遥感影像
  │
  ▼ SpectralAttention: 通道重标定
  [B, 4, 512, 512]
  │
  ▼ Encoder (ResNet50):
  ┌───────────┬──────────┬───────────┬──────────────────────┐
  │ Stage     │ H×W      │ 通道      │ 跳连到               │
  ├───────────┼──────────┼───────────┼──────────────────────┤
  │ conv1+pool│ 128×128  │ 64        │ → decoder1           │
  │ layer1    │ 64×64    │ 256       │ → decoder2           │
  │ layer2    │ 32×32    │ 512       │ → decoder3           │
  │ layer3    │ 16×16    │ 1024      │ → decoder4           │
  │ layer4    │ 8×8      │ 2048      │                      │
  └───────────┴──────────┴───────────┴──────────────────────┘
  │
  ▼ Bridge: Conv3×3 → BN → ReLU
  [B, 256, 8, 8]
  │
  ▼ Decoder (每个Block: Up → Cat → Conv → CBAM → Conv → Drop):
  ┌────────┬────────────┬───────────┬──────────────────────┐
  │ Block  │ 输入+跳连   │ H×W       │ 输出通道              │
  ├────────┼────────────┼───────────┼──────────────────────┤
  │ dec4   │ 8+16=16    │ 16×16    │ 128                  │
  │ dec3   │ 16+32=32   │ 32×32    │ 64                   │
  │ dec2   │ 32+64=64   │ 64×64    │ 32                   │
  │ dec1   │ 64+128=128 │ 128×128  │ 32                   │
  └────────┴────────────┴───────────┴──────────────────────┘
  │
  ▼ 三任务头 (在dec1的32ch特征之上):
  │
  ├─ BoundaryHead: [B, 1, 256, 256]  ← 上采样到H/2
  │
  ├─ DistanceHead: [B, 1, 256, 256]  ← Sigmoid归一化到[0,1]
  │
  └─ SegUp: [B, 16, 256, 256]        ← 上采样到H/2
       │
       ▼ Fusion: Cat(16+1+1=18ch) → Conv3×3→16 → BN+ReLU+Drop → Conv1×1→1
       [B, 1, 256, 256]  ← 最终分割logits
       │
       ▼ 后处理: Sigmoid(>0.5) → 二值掩膜
       [B, 1, 256, 256]  ← 耕地/非耕地
```

## 12. V7 → V8 核心演进

| 特性 | V7 | V8 | 原因 |
|------|-----|-----|------|
| 输入通道 | 3 (RGB) | **4 (R+G+B+NIR)** | 遥感影像通常有近红外波段 |
| 光谱处理 | 无 | **SpectralAttention**入口通道重标定 | 让模型自动学习各通道重要性 |
| 首层卷积 | 直接3通道 | **自适应扩展到4通道** | 保留ImageNet预训练权重 |
| 主损失函数 | DiceBCE | **LogCoshDice + Focal** | 更平滑的梯度 + 专注难样本 |
| 可选损失 | 无 | **TverskyLoss** | 处理边界不平衡场景 |
| 优化器 | Adam | **AdamW** | 解耦权重衰减，泛化更强 |
| 梯度裁剪 | 无 | **max_norm=1.0** | 防止梯度爆炸 |
| Warmup | 无 | **LinearLR warmup** (2-3 epoch) | 初期稳定训练 |
| V8_4090特化 | 无 | batch=32, lr=5e-4, workers=8 | 针对RTX 4090优化 |

### V8 vs V8_4090

两个文件**神经网络架构完全一致**，区别仅在训练配置：

| 配置 | V8 | V8_4090 |
|------|-----|---------|
| 默认batch_size | 16 | 32（4090显存更大） |
| 默认lr | 1e-4 | 5e-4（更大batch可配更大lr） |
| 默认epochs | 50 | 100 |
| num_workers | 0 | 8 |
| dropout_rate | 0.2 | 0.1（更多数据可降低dropout） |
| warmup_epochs | 2 | 3 |
| early_stopping | 8 | 12 |
| weight_decay | 1e-4 | 5e-5 |

## 13. 与Field-Net论文的对应关系

| Field-Net论文设计 | V8实现 | 说明 |
|-------------------|--------|------|
| U-Net + 残差块编码器 | ResNet50/34/18编码器（残差结构） | 更成熟的残差网络 |
| 空间注意力模块 | **CBAM**（通道+空间双重注意力） | V8升级为双重注意力 |
| 多任务：掩膜+边界+距离 | **三头输出 + Fusion融合** | 完全继承并改进 |
| 3×3 Conv + BN残差块 | DecoderBlock: Conv3×3 + BN + ReLU + Dropout | 增加了Dropout正则化 |
| Dice loss + MAE | LogCoshDice + DiceBCE + L1 + Focal | 更丰富的损失组合 |
| Adam优化器 | AdamW + Warmup + 梯度裁剪 | 更先进的优化策略 |
| 数据增强 | 相同 + ChannelShuffle + BrightnessContrast | 更多样的增强 |
| 输入256×256×4 RGBN | 支持512×512×4 | 更高分辨率的输入 |

### V8相对于Field-Net论文的主要改进：

1. **CBAM（双重注意力）**：Field-Net仅使用空间注意力，V8同时使用通道+空间注意力
2. **SpectralAttention前置光谱校准**：在输入层增加光谱通道重标定
3. **LogCoshDice + Focal + Tversky损失体系**：受PRUE论文启发
4. **AdamW + 梯度裁剪 + Warmup**：更稳健的训练策略
5. **Dropout正则化**：在每个DecoderBlock中嵌入Dropout
